> Methodology: each page is measured as a **warm SPA navigation** from the dashboard — the cost of *arriving on that page* with the wasm runtime already booted. Metrics come from the browser's own Performance timeline (long-task, layout-shift, paint observers), not source instrumentation. The one-time wasm boot is scored separately under **Cold load**. CPU-time metrics (mount/TBT/settle) use **best-of-11** — OS-scheduler contention can only add time, so the fastest pass reflects intrinsic render cost; CLS/DOM use the median.
